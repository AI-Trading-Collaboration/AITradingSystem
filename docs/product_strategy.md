# 产品策略

## 系统定位

本项目定位为：

```text
AI 产业链投资认知与仓位决策支持系统
```

不是：

```text
AI 自动炒股系统
```

系统最重要的目标不是预测明天涨跌，而是把交易认知拆成可验证、可复盘、可迭代的模块。

长期升级目标是可审计认知模型：系统持续更新对 AI 产业链状态、风险来源、thesis 有效性、判断置信度和仓位边界的认知，但不直接自动交易，也不允许未经验证的 LLM 或新闻结论直接改变生产仓位规则。该目标的工程拆解见 `docs/requirements/cognitive_model_2026-05-04.md`。

核心闭环：

```text
信息输入 -> 产业链映射 -> 假设更新 -> 风险评估 -> 仓位决策 -> 复盘归因
```

规划中的认知状态闭环会在假设更新、风险评估和仓位决策之间显式保存 `belief_state`：

```text
证据 / thesis / risk / valuation / industry node
-> belief_state
-> 评分 / 置信度 / 仓位边界 / 报告 / 校准
```

默认市场阶段是 ChatGPT 后 AI 主线行情。工程配置中的 `ai_after_chatgpt` 锚定 `2022-11-30` ChatGPT 公开发布，默认回测和策略解释从 `2022-12-01` 开始。更早数据可用于 warm-up、压力测试和跨周期比较，但不应替代这个默认结论窗口。

## 能力圈

系统优先服务于技术理解能够形成优势的方向：

- AI 算力链。
- GPU / ASIC。
- 半导体制造。
- 先进封装。
- HBM。
- 云厂商 CapEx。
- XR / 空间计算。
- 图形渲染和底层基础设施。

能力圈之外的交易不应被系统当作同等置信度处理。后续持仓和观察池模块需要记录每个标的是否在能力圈内，以及能力圈置信度。

## 产业链因果图

AI 半导体链条应按因果关系建模，而不是只按新闻热度排序：

```text
云厂商 CapEx
-> GPU/ASIC 需求
-> HBM 需求
-> 先进封装需求
-> 晶圆代工需求
-> 设备与材料需求
```

每条信息都应尽量回答：

- 它影响哪个产业链节点？
- 它改变短期情绪，还是长期现金流预期？
- 它是领先指标、同步指标，还是滞后指标？
- 它增强、削弱，还是不改变原始投资假设？

## 收益归因

系统需要避免把行情 Beta 误判成自身 Alpha。后续复盘模块应尽量拆分：

```text
收益 = 市场 Beta + 行业 Beta + 主题 Beta + 个股 Alpha + 运气
```

第一版可以先用 SPY、QQQ、SMH/SOXX 作为归因基准，后续再加入更完整的因子和组合归因。

## 仓位原则

系统输出必须把观点转换成仓位，而不是停留在“看好/不看好”。

仓位应由多个维度共同折扣：

```text
目标仓位 = 产业确定性 * 估值性价比 * 市场环境 * 风险事件折扣 * 技术面确认
```

关键原则：

- 看好不等于满仓。
- 看空不等于清仓。
- 产业强但估值过热时应限制仓位。
- 基本面强但宏观流动性恶化时应折扣仓位。
- 事件风险高时应单独控制暴露。

## 反身性和拥挤度

系统必须持续区分：

```text
好公司 != 好价格
好产业 != 随时可以买
```

后续估值与预期模块应跟踪：

- 估值分位数。
- 分析师一致预期是否过度乐观。
- 财报后利好不涨。
- 期权隐含波动率。
- 媒体和散户热度。
- ETF 资金流入集中度。
- 龙头股是否开始跑输。
- 市场是否从业绩驱动转为情绪驱动。

## 风险事件分级

风险事件需要结构化，避免主观恐慌交易。

|等级|含义|仓位动作|
|---|---|---|
|L1|普通噪音，例如普通负面报道、个别评级下调|观察，不轻易调整|
|L2|中等风险，例如利率快速上行、CapEx 表态谨慎、出口限制升级|降低部分仓位，等待确认|
|L3|重大风险，例如战争升级、台海风险、AI CapEx 逻辑证伪、龙头财报全面低于预期|快速降风险，保留现金，等待重新定价|

## 假设验证

每笔重要交易都应记录 thesis：

- 买入理由。
- 验证指标。
- 证伪条件。
- 目标周期。
- 仓位理由。
- 风险事件处理规则。

系统定期检查：

```text
原始假设是否仍然成立？
```

如果交易赚钱，也要区分是判断正确、市场 Beta、仓位运气，还是规则外行为。

## 核心模块优先级

|优先级|模块|回答的问题|
|---|---|---|
|P0|持仓与观察池管理|我正在观察/持有什么，为什么，风险等级是什么？|
|P1|产业链信息监控|产业趋势有没有变？|
|P2|市场环境|当前适不适合承担风险？|
|P3|估值与预期|好公司现在是不是好价格？|
|P4|仓位决策|应该加仓、减仓、持有还是等待？|
|P5|复盘归因|这次赚钱/亏钱的真正原因是什么？|

## 关键设计原则

- 把技术优势转化为产业链判断。
- 把观点转化为仓位规则。
- 把每次交易变成可复盘数据。
- 让系统持续回答：在我理解的产业趋势中，当前市场价格是否给了足够好的风险收益比？
