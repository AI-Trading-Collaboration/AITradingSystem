# 日报生产就绪缺口登记

状态：IN_PROGRESS

最后更新：2026-05-04

关联任务：`PROD-001`、`RISK-005`、`DATA-001`、`PORTFOLIO-002`、`SCORE-003`、`COST-001`、`BACKTEST-001`、`GOV-001`

## 背景

2026-05-04 日报生成成功，但结论使用等级为 `data_limited`，执行动作是 `observe_only`。这说明当前报告可以作为人工投研复核输入，但还不能直接作为生产交易依据。

## 生产化缺口与处理顺序

|顺序|缺口|承接任务|当前判断|
|---:|---|---|---|
|1|风险事件发生记录缺少“今日已复核且未发现未记录重大事件”的可审计输入，空目录不能证明没有政策/地缘风险|`RISK-005`|工程底座已完成基础版；真实复核 owner、来源清单和运行纪律仍需生产前落实|
|2|估值和盈利预期数据仍受 point-in-time 样本、历史覆盖和 `eps_revision_90d_pct` 覆盖限制|`DATA-001`、`BTINPUT-001`|受数据供应商和样本积累限制，不能用采集日快照伪造历史可见性|
|3|日报仓位仍不是账户级建议，缺少真实账户持仓输入、现金/税费/流动性扩展和风险预算联动|`PORTFOLIO-002`|工程底座已完成基础版：真实持仓 CSV、只读暴露报告和日报 `NOT_CONNECTED` 边界已具备；生产前仍需 owner 提供真实持仓并继续由 `SCORE-003` 接入风险预算|
|4|固定仓位区间与硬闸门不足以表达目标波动率、最大回撤、相关性、节点集中度和估值拥挤的综合约束|`SCORE-003`|工程底座已完成基础版：`risk_budget` gate 已接入市场压力和真实组合集中度约束；生产前仍需真实持仓、参数 replay/shadow、流动性/税费和完整相关性预算验证|
|5|执行层仍是 advisory，交易成本、滑点、订单拆分、冷却期和人工审批还没有完整生产闭环|`COST-001`、`EXEC-001`|工程底座已完成基础版：执行动作词表、成本假设拆分、回测明细/报告和 trace 成本参数已具备；生产前仍需真实成交样本、券商/税务规则、容量约束和人工审批链|
|6|回测和校准还不足以支撑生产信任，需要抗过拟合、参数敏感性、样本外和 baseline 对比|`BACKTEST-001`|工程底座已完成基础版：`aits backtest` 可选输出稳健性报告，覆盖成本压力、起点后移和买入持有基准；完整结论仍依赖历史风险/估值输入覆盖、权重扰动、随机基线和样本外验证|
|7|规则治理仍停留在 baseline rule card registry，缺少正式 promotion / retirement / owner approval 与规则版本注入|`GOV-001`|可继续补版本注入和审批台账，但生产上线仍需 owner 批准流程|

## 当前优先工作

第一步已推进 `RISK-005`。目标不是让系统自动认定“没有风险”，而是增加一条可审计的人工复核声明路径：只有当复核声明覆盖评估日、来源范围完整、复核人和复核结论明确、且未过期时，政策/地缘模块才可把“没有可评分 active 风险事件”作为经过复核的输入处理。
