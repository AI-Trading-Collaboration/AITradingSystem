# 产物目录

本文从“看到一个文件后怎么理解它”的角度列出关键 artifact。`docs/system_flow.md` 仍是工程事实源；本文是阅读索引，帮助使用者判断 artifact 的生成者、上游输入、下游用途、production 影响和常见误解。

如果需要理解输入数据如何计算成输出数据，先读 `docs/calculation_logic.md`；字段级含义见 `docs/schema/fields.yaml`。该 YAML 先覆盖 `scores_daily.csv`、decision snapshot、trace bundle、prediction ledger 和 shadow parameter search 的核心字段。

## ETF Portfolio runtime artifact policy

`data/etf_portfolio/`、`data/simulation/` 和 `reports/` 是本地运行产物目录，默认由 ETF daily run、simulation、backtest、P1/P2 observe-only reports 或 report integration 命令生成，并通过 `.gitignore` 排除在源码提交之外。它们可以作为本机审计证据和人工复核输入，但不是 source artifacts。

ETF deterministic test fixtures 只能提交到 `tests/fixtures/etf_portfolio/`。不要把测试 fixture 放进 `data/` 或 `reports/`，也不要把 daily run 生成的 report 复制成源码 fixture，除非另行建立明确的最小化、脱敏、确定性 fixture。

TRADING-064 controlled calibration experiment definitions 是 source configuration，位于 `config/etf_portfolio/experiments.yaml` 和 `config/etf_portfolio/experiment_packs.yaml`。这些文件只定义允许观察的 experiment registry、`etf_calibration_v1` pack、`risk_adjusted_v1` ranking policy、`shadow_only_manual_review` candidate gate policy、`weekly_shadow_review_v1` review policy 和安全边界，不是 runtime 结果；batch run 后的 `comparison_report.json/md` 和 `candidate_selection_report.json/md` 生成在 `reports/etf_portfolio/experiments/<run_id>/`，只能作为 shadow/manual-review 输入。`aits etf experiments enroll-shadow` 生成 `data/simulation/etf_shadow_candidates.json`，登记 `eligible_for_shadow` candidate 的 `shadow_id`、model/config hash、source run 和 evaluation schedule；`aits etf experiments weekly-review` 生成 `reports/etf_portfolio/experiments/weekly_reviews/weekly_review_YYYY-MM-DD.json/md`，只读比较 candidate/baseline/QQQ、drawdown、turnover、constraint hits 和 recommended action；`aits etf experiments validate --pack etf_calibration_v1` 生成 `reports/etf_portfolio/experiments/validation/*_experiment_validation.json/md`，作为 TRADING-064 final safety gate。上述文件都在 ignored runtime 目录，不是 source artifact，并保留 `observe_only=true`、`production_effect=none`、`broker_action=none`、`manual_review_required=true`、`production_promotion_allowed=false`。这些 runtime artifacts 已登记到 `config/report_registry.yaml`，由 `aits reports index` 和 Reader Brief 只读发现；缺失时只能显示 `MISSING`，不得补跑 experiment 或补造 candidate 状态。

TRADING-065 forward simulation policy 是 source configuration，位于 `config/etf_portfolio/forward_simulation.yaml`。该文件治理 rolling windows、candidate lifecycle thresholds、allowed statuses、allowed review/watchlist actions 和 safety boundary；它是 pilot baseline policy，不是 production allocation policy。`aits etf forward update --date YYYY-MM-DD` / `--latest` 只读取 shadow candidate registry、ETF price cache、baseline config 和 benchmarks，写入 evaluation-only update report、更新 shadow registry 的 evaluation metadata，并把 decision-time rows 追加到 `data/simulation/etf_forward_decisions.csv`；decision ledger 不得包含 `forward_*` 或其他 future/evaluation fields。`aits etf forward dashboard --latest`、`aits etf forward weekly-review --latest`、`aits etf forward watchlist --latest` 和 `aits etf forward validate` 分别生成 dashboard、weekly review、local watchlist 和 final validation gate。所有输出都在 ignored runtime 目录，固定 `observe_only=true`、`production_effect=none`、`broker_action=none`、`manual_review_required=true`、`production_promotion_allowed=false`，不得写 official target weights、替换 baseline 或触发 broker。Forward artifacts 已登记到 `config/report_registry.yaml`，daily ops 会在 report index 和 Reader Brief 之前刷新 update/dashboard/watchlist；Reader Brief 只读展示 `ETF Forward Simulation` 摘要。

TRADING-066 AI confirmation universe 是 source configuration，位于 `config/etf_portfolio/ai_confirmation_universe.yaml`。该文件定义 mega-cap AI、semiconductor hardware、cloud AI platform、AI ETF proxy 和 event-risk reference groups 的 ticker/name/group/role/enabled/weight_cap/benchmark/data_required/optional metadata，并固定 `observe_only=true`、`candidate_only=true`、`production_effect=none`、`broker_action=none`、`manual_review_required=true`。`config/etf_portfolio/ai_confirmation_policy.yaml` 治理 score bands、MegaCapAIScore component weights、AI semiconductor relative-strength pair definitions/weights、event risk severity/bands、relative-strength normalization、drawdown penalty、event-risk adjustment、composite weights、shadow overlay thresholds/constraints 和 coverage warning floor。`aits etf ai-confirmation features --date YYYY-MM-DD` 生成 `reports/etf_portfolio/ai_confirmation/features/ai_confirmation_features_YYYY-MM-DD.json/csv`，只包含 price-derived breadth features、data coverage 和 warnings；运行前必须通过 ETF price quality gate，strict required universe data 缺失时 fail closed。`MegaCapAIScore`、`AISemiconductorRelativeStrengthScore`、event risk overlay 和 `AIConfirmationScore` composite 先作为 report 前的内存 payload，包含 score/risk value、band、component_scores or active/upcoming/recent events、drivers/reason codes、action_hint、coverage or pair features、policy version/hash 和 safety fields。`aits etf ai-confirmation report --date YYYY-MM-DD` 生成 `reports/etf_portfolio/ai_confirmation/reports/ai_confirmation_report_YYYY-MM-DD.json/md`，汇总 safety banner、AIConfirmationScore、component score table、semiconductor breadth、mega-cap AI confirmation、AI/semiconductor relative strength、event risk、data coverage、top drivers、candidate-only usage note 和 shadow experiment usage note；该 report 已登记到 `config/report_registry.yaml`，由 `aits reports index` 只读发现。`aits etf ai-confirmation overlay --date YYYY-MM-DD --candidate <candidate_id>` 只读取显式 base candidate weights 和 AI confirmation report，输出 bounded candidate-only `after_candidate_weights`、`candidate_weights`、`shadow_weights` 和 `hypothetical_weights` 到 ignored overlay artifacts。它不产生 production target weights，也不替换 TRADING-064/065 的 shadow candidate 或 forward artifacts；后续 validation artifacts 必须从该 config/feature/report/overlay output 读取 universe membership、coverage 和 safety fields。

TRADING-067 satellite stock replacement policy 是 source configuration + runtime artifact 组合。`config/etf_portfolio/satellite_universe.yaml` 定义 AI / semiconductor 个股、benchmark ETF、sleeve、role、optional flag、data coverage 和 event risk group；`config/etf_portfolio/satellite_policy.yaml` 治理 score bands、component weights、eligibility thresholds、single-name/sleeve/residual ETF/risk constraints、validation evidence 和 review condition。`aits etf satellite features --date YYYY-MM-DD` 生成 `reports/etf_portfolio/satellite/features/satellite_features_YYYY-MM-DD.json/csv`，只使用 `date <= score_date` 的 stock/benchmark ETF 价格计算 relative strength、trend、momentum、drawdown、volatility 和 coverage features。`aits etf satellite report --date YYYY-MM-DD` / `run` 生成 `reports/etf_portfolio/satellite/reports/satellite_replacement_report_YYYY-MM-DD.json/md`，包含 universe coverage、SatelliteCandidateScore、replacement eligibility gate、candidate/shadow/hypothetical replacement plan、risk constraint diagnostics、AI confirmation context 和 safety fields；`aits etf satellite experiment --date YYYY-MM-DD` 生成 ignored runtime shadow experiment；`aits etf satellite validate` 生成 `reports/etf_portfolio/satellite/validation/satellite_validation_YYYY-MM-DD.json/md`。这些 artifacts 已登记到 `config/report_registry.yaml`，Reader Brief 只读展示 `Satellite Replacement` 摘要。默认 ETF first；数据不足、gate fail 或风险约束触发时输出 `fallback_to_etf=true`，不得写 official ETF target weights、不得触发 broker action、不得自动 promotion。

TRADING-068 ETF weekly portfolio review 是 runtime 汇总报告，不新增 production allocation policy。`aits etf weekly-review aggregate --as-of YYYY-MM-DD` 只读扫描 `config/report_registry.yaml` / optional report index 指向的 ETF brief、target weights、experiment weekly review、forward dashboard/watchlist、AI confirmation、satellite replacement 和 validation gate artifacts，输出 `reports/etf_portfolio/weekly_review/aggregation/weekly_review_aggregation_YYYY-MM-DD.json`。`aits etf weekly-review generate --as-of YYYY-MM-DD` 生成 `reports/etf_portfolio/weekly_review/weekly_review_YYYY-MM-DD.json/md`，集中 portfolio decision summary、shadow candidate review、AI confirmation review、satellite replacement review、risk/watchlist/constraints、manual review action items、source report links、validation status 和 next-week watch items。`aits etf weekly-review validate` 生成 `reports/etf_portfolio/weekly_review/validation/weekly_review_validation_YYYY-MM-DD.json/md`，确认 report generator、Reader Brief integration、source traceability、unsafe action blocking 和 safety fields。所有 weekly review 输出固定 `observe_only=true`、`candidate_only=true`、`production_effect=none`、`broker_action=none`、`manual_review_required=true`；缺失可选 report 只能显示 `missing_data` / `REPORT_NOT_FOUND`，不得补跑上游、补造结论、写 official target weights 或触发 broker。

TRADING-069 ETF portfolio decision journal 是人工复核后的 runtime state + report。`aits etf decision-journal add/update/list/remove` 维护 ignored state `data/simulation/etf_portfolio_decision_journal.json`，每个 active entry 必须链接 TRADING-068 weekly review JSON 和 `manual_review_actions[].action_id`，并保留 source section、action snapshot、source evidence、rationale、confidence、follow-up task 和 audit trail。`remove` 只把 entry 移到 `removed_entries`，不静默删除。`aits etf decision-journal report --as-of YYYY-MM-DD` 生成 `reports/etf_portfolio/decision_journal/decision_journal_YYYY-MM-DD.json/md/html`，`analytics` 生成 review outcome JSON，`propose-state-updates` 只生成 candidate state proposal，`validate` 生成 `reports/etf_portfolio/decision_journal/validation/decision_journal_validation_YYYY-MM-DD.json/md`。所有 journal artifacts 固定 `observe_only=true`、`candidate_only=true`、`production_effect=none`、`broker_action=none`、`manual_review_required=true`；proposal 不修改 shadow registry、production weights 或 broker state。

TRADING-070 ETF allocation parameter review 聚合 forward evidence，不新增 production allocation policy。`aits etf parameter-review aggregate --as-of YYYY-MM-DD` 只读扫描 `config/report_registry.yaml` / optional report index 指向的 forward dashboard、forward weekly review、watchlist、TRADING-068 weekly review、TRADING-069 decision journal、experiment comparison、candidate selection 和 validation gates，输出 `reports/etf_portfolio/parameter_review/aggregation/parameter_review_evidence_YYYY-MM-DD.json/md`。Aggregation payload 保留 source report paths、candidate evidence records、metrics、journal/weekly links、validation status、missing source warnings 和 safety banner；缺少 forward dashboard 或 forward candidate rows 时输出 `needs_more_data` / `INSUFFICIENT_FORWARD_EVIDENCE`，缺失可选 source 不允许用补造结论解决。`aits etf parameter-review report --as-of YYYY-MM-DD` / `run` 生成 `reports/etf_portfolio/parameter_review/reports/parameter_review_YYYY-MM-DD.json/md`，把 evidence aggregation、candidate comparison、journal linkage、proposal package、governance scorecard、manual review requirements、next steps 和 source links 汇总为人工 review package。`aits etf parameter-review validate` 生成 `reports/etf_portfolio/parameter_review/validation/parameter_review_validation_YYYY-MM-DD.json/md`，确认 schema、aggregator、comparison、journal linker、proposal generator、governance gate、report generator、Reader Brief visibility、source links 和 unsafe action blocking。Reader Brief 的 `ETF Parameter Review` 区块只读 latest `etf_parameter_review_report`，展示 status、candidate/proposal counts、main reason、safety posture 和 detail link；缺失时显示 `MISSING`，不补跑上游。所有 parameter review artifacts 固定 `observe_only=true`、`candidate_only=true`、`production_effect=none`、`broker_action=none`、`manual_review_required=true`；当前阶段只建立 evidence/proposal package，不修改 baseline config、production weights、shadow registry 或 broker state。

TRADING-071 ETF allocation dual-track weight calibration 先建立 historical candidate search，不新增 production allocation policy。`config/etf_portfolio/weight_search.yaml` 是 source configuration，定义 `etf_initial_weight_search_v1` 的 universe、5% bounded grid、candidate cap、objective policy、benchmark set、walk-forward windows、regime split labels 和固定 safety boundary。`aits etf weight-calibration search --search etf_initial_weight_search_v1` 先通过 ETF price quality gate，再枚举有效 grid 并按 config 上限选择 deterministic bounded subset，生成 `reports/etf_portfolio/weight_calibration/<run_id>/summary.json/md`、`metrics.csv`、`ranking.json`、`robustness.json` 和 `data/etf_portfolio/weight_calibration/<run_id>/candidate_weight_sets.json/csv`。Search payload 披露 total valid candidate count、evaluated candidate count、baseline weights、benchmark metrics、HistoricalWeightScore components、walk-forward/regime slice metrics、stability score、hard blockers、blocked candidates、market regime 和 requested date range。`aits etf weight-calibration register-candidates --run-id/--latest --top N` 把 selected historical candidates 写入 ignored `data/etf_portfolio/weight_calibration/candidate_weight_registry.json`，保留 `weight_set_id`、source search run、rank、weights、metrics summary、robustness summary、blockers、selection reason、config hash、status 和 safety fields；重复登记同一 `weight_set_id` 为 idempotent update。`aits etf weight-calibration enroll-forward --weight-set <id>` 或 `--latest --top N` 只把 safe candidate 登记到 ignored `data/etf_portfolio/weight_calibration/forward_enrollments.json`，生成 `shadow_id`、nested shadow-style record、`forward_tracking_link`、tracking state、historical source linkage、metrics/robustness snapshot 和 safety fields；blocked/rejected/needs_more_data candidates 不能 enroll。`aits etf weight-calibration aggregate-evidence --as-of YYYY-MM-DD` 读取 candidate registry、forward enrollment ledger、optional historical search summary 和 optional forward/weekly/journal/parameter-review JSON，输出 `reports/etf_portfolio/weight_calibration/evidence/backtest_forward_evidence_YYYY-MM-DD.json/md`；每个 record 链接 `weight_set_id`、`shadow_id` 和 source search run，计算 `expectation_gap`、`drawdown_gap`、`turnover_gap`、`stability_gap`，并输出 `consistent`、`forward_better_than_backtest`、`forward_worse_than_backtest`、`mixed`、`blocked` 或 `needs_more_forward_data`。缺少 forward row 或 forward days 不足时必须保持 `needs_more_forward_data`，不得补造 forward conclusion。`aits etf weight-calibration overfit-diagnostics` 读取 candidate registry、optional search summary 和 optional TRADING-071F evidence，输出 `reports/etf_portfolio/weight_calibration/overfit_diagnostics/overfit_diagnostics_*.json/md`；component diagnostics 覆盖 performance concentration、single-period dependency、regime fragility、turnover instability、constraint hit instability、weight extremeness、benchmark dependency 和 forward/backtest divergence，并输出 low/medium/high/critical risk band。`aits etf weight-calibration generate-proposals` 读取 candidate registry、optional TRADING-071F evidence 和 optional TRADING-071G diagnostics，输出 `reports/etf_portfolio/weight_calibration/proposals/candidate_weight_proposals_*.json/md`；proposal types 只允许 `continue_forward_observation`、`reject_weight_set`、`defer_until_more_forward_data`、`propose_extended_shadow` 和 `propose_manual_baseline_review`，并显式阻断 `apply_weight_set`、`promote_to_production`、`enable_broker_action`。该 ledger 是 TRADING-071 weight-calibration forward observation state，不写 shared experiment shadow registry `data/simulation/etf_shadow_candidates.json`，不替换 forward dashboard 的 experiment-backed candidates。所有输出固定 `observe_only=true`、`candidate_only=true`、`production_effect=none`、`broker_action=none`、`manual_review_required=true`，并记录 `production_weights_mutated=false` / `applied_weight_set=null`；candidate weights 只是后续 report/manual review 输入，不写 `target_weights.csv`、不修改 baseline config、不触发 broker action。

ETF no-lookahead artifacts 遵守统一 timing contract：raw market data / feature snapshot / signal / allocation decision 使用同一 `t`，最早 execution date 是 `t` 之后的下一交易日，backtest return date 必须晚于 execution date。`data/simulation/etf_ledger.csv` 中的 `forward_return_*`、`relative_return_vs_*`、`max_drawdown_next_*`、`weight_contribution_*`、portfolio-vs-benchmark 和 `signal_hit_*` 字段属于 delayed evaluation，只能在 `evaluation_only=true` 记录中承载；ETF daily brief 的 decision sections 不得引用这些字段。

## production_effect 标签

| 标签 | 含义 | 典型位置 |
|---|---|---|
| `production` | 正式生产判断链路的一部分，会进入 canonical scoring / snapshot / ledger 语义 | production decision snapshot、production prediction ledger 行 |
| `advisory` | 投研辅助或趋势判断输出，不自动交易 | daily score report、execution advisory |
| `none` | 只读、诊断、shadow 或 validation-only，不改变生产评分、仓位 gate、正式 ledger 或 approved overlay | dashboard、shadow search、flow validation ledger |
| `validation-only` | 用于验证参数、规则或流程，不可当成生产候选批准 | parameter search、shadow gate / weight profiles |
| `blocked` | 被质量、治理、样本、owner approval 或 promotion contract 阻断 | promotion report、candidate governance |

运行时核心标签由 `ai_trading_system.core.production_effect.ProductionEffect` 表达。历史报告中较长的解释性 production effect 文本需要逐处确认语义后再迁移，不能自动归并。

当一个产物声明 `production_effect=none` 时，它仍然可以非常有价值，但价值是诊断、学习、复核或准备 forward shadow，而不是直接改变 production 结论。
